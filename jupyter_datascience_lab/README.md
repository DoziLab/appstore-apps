# DoziLab Jupyter Data Science Lab

Diese App provisioniert eine Ubuntu-VM pro Kurs/Stack mit JupyterHub,
JupyterLab und typischen Python/Data-Science-Paketen. Sie legt lokale
Kursbenutzer an und stellt JupyterHub ueber Nginx auf Port 80 bereit.

Wichtig fuer V1: Diese App erstellt bewusst keine Cinder Volumes. Die VM bootet
direkt vom ausgewaehlten Image; Userdaten liegen unter `/home/<user>`.

## Ueberblick

Die App erstellt:

- 1 Ubuntu VM
- Floating IP aus dem DHBW-Netz
- SSH-Zugriff nur aus `ssh_cidr`
- JupyterHub Webzugriff nur aus `web_cidr`
- lokale Linux-User aus `user_json`
- JupyterLab pro User mit eigenem Home-/Work-Verzeichnis
- SSH nur key-basiert fuer den `ubuntu` Support-User
- optionale Beispiel-Notebooks
- READY/FAILED Marker im Nova Console Log

## Backend-Parameter

- `stack_label` (string, required)
- `image` (string, required)
- `flavor` (string, required)
- `network` (string, fixed: `NAT`)
- `external_network` (string, fixed: `DHBW`)
- `key_name` (string, fixed: `heat-bastion-key`)
- `ssh_cidr` (string, required)
- `web_cidr` (string, required)
- `install_examples` (boolean, required)
- `workdir` (string, required)
- `user_json` (string, generated by backend)

## user_json Schema

Beispiel, unencoded:

```json
{
  "course_label": "Data Science Kurs 2026",
  "instance": {
    "credentials": [
      { "username": "gruppe01", "password": "User01_2026!" },
      { "username": "gruppe02", "password": "User02_2026!!" }
    ],
    "admin_credentials": {
      "username": "teacher",
      "password": "TeacherPw_2026!"
    }
  },
  "applications": [
    { "name": "jupyterhub", "version": "1.0.0" }
  ]
}
```

`user_json` kann Base64-kodiertes UTF-8 JSON oder Raw-JSON sein.

## Zugriff

### JupyterHub

```text
http://<floating_ip>/
```

Login mit `username` und `password` aus `instance.credentials` oder
`instance.admin_credentials`. Der User aus `instance.admin_credentials` wird in
JupyterHub als Admin-User gesetzt.

### SSH Admin

```bash
ssh -i ~/.ssh/heat-bastion-key.pem ubuntu@<floating_ip>
```

Passwort-SSH fuer Kursbenutzer ist bewusst deaktiviert. Die Kursbenutzer melden
sich ueber JupyterHub an.

## Ready/Fail Marker

Bei Erfolg:

```text
DOZILAB_READY stack=<label> time=<timestamp>
```

Datei: `/var/lib/dozilab/ready`

Bei Fehler:

```text
DOZILAB_FAILED stack=<label> rc=<code> time=<timestamp>
```

Datei: `/var/lib/dozilab/failed`
